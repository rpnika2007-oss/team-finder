# TeamFinder

TeamFinder — веб-приложение для поиска команды в pet-проекты. Пользователи могут регистрироваться, создавать проекты, присоединяться к проектам других участников, добавлять проекты в избранное, указывать навыки и фильтровать участников/проекты по навыкам.

Проект выполнен на Django с использованием готового фронтенда Яндекс Практикума и backend-логики для варианта 2.

## Реализованный вариант

**Вариант 2 — «Навыки», фильтрация участников по навыкам.**

В проекте реализованы:

* регистрация пользователей;
* вход и выход из аккаунта;
* кастомная модель пользователя с авторизацией по email;
* профиль пользователя;
* редактирование профиля;
* смена пароля;
* создание проекта;
* редактирование проекта;
* завершение проекта;
* участие в проекте;
* отказ от участия;
* добавление проектов в избранное;
* просмотр избранных проектов;
* добавление навыков пользователю;
* удаление навыков пользователя;
* автодополнение навыков;
* создание нового навыка из интерфейса;
* фильтрация пользователей по навыкам;
* фильтрация проектов по необходимым навыкам;
* кнопка «Поделиться профилем»;
* кнопка «Поделиться ссылкой» на проект;
* админ-панель для управления пользователями, проектами и навыками.

## Переменные окружения

В проекте используется файл `.env`.

Файл `.env` не должен попадать в GitHub. Для примера настроек используется файл `.env_example`.

Пример `.env_example`:

```env
DJANGO_SECRET_KEY=replace-me
DJANGO_DEBUG=True

POSTGRES_DB=teamfinder
POSTGRES_USER=teamfinder
POSTGRES_PASSWORD=teamfinder
POSTGRES_HOST=127.0.0.1
POSTGRES_PORT=55432

TASK_VERSION=2
```

Пояснение:
В проекте используется порт `55432`, чтобы избежать конфликта с локальным PostgreSQL на стандартном порту `5432`.

## Запуск проекта в Windows CMD

Перейдите в папку проекта;
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
copy .env_example .env

Проверьте, что в `.env` указаны параметры:

DJANGO_SECRET_KEY=replace-me
DJANGO_DEBUG=True

POSTGRES_DB=teamfinder
POSTGRES_USER=teamfinder
POSTGRES_PASSWORD=teamfinder
POSTGRES_HOST=127.0.0.1
POSTGRES_PORT=55432

TASK_VERSION=2


Запустите PostgreSQL через Docker:

docker compose up -d db

Проверьте контейнер:

docker compose ps

В выводе должен быть проброс порта:

55432->5432/tcp

Проверьте подключение к PostgreSQL:

python -c "import psycopg; psycopg.connect(dbname='teamfinder', user='teamfinder', password='teamfinder', host='127.0.0.1', port='55432'); print('OK')"

Если всё настроено правильно, команда выведет:
OK

Создайте миграции:
python manage.py makemigrations users projects

Примените миграции:

python manage.py migrate

Запустите сервер:

python manage.py runserver

Откройте сайт:

http://127.0.0.1:8000/

Корневой путь перенаправляет на список проектов:
http://127.0.0.1:8000/projects/list/

## Тестовый администратор

В проекте можно использовать администратора:

email: maria@yandex.ru
password: password

Если такого пользователя нет, его можно создать вручную:

python manage.py createsuperuser

При создании укажите:

Email: maria@yandex.ru
Имя: maria
Фамилия: sap
Password: password

Админ-панель доступна по адресу:

http://127.0.0.1:8000/admin/

Администратор имеет доступ к управлению:

* пользователями;
* проектами;
* навыками;
* участниками проектов;
* избранными проектами;
* статусами проектов.

Права администратора определяются стандартными полями Django:

is_staff=True
is_superuser=True
is_active=True

## Навыки пользователей

На странице профиля отображается блок «Навыки».

Владелец профиля может:

* добавить навык;
* выбрать навык из автодополнения;
* создать новый навык;
* удалить навык из своего профиля.

Другие пользователи и гости могут только просматривать навыки.


Если навык содержит специальные символы, например `c++`, он передаётся в URL в закодированном виде:

/users/list/?skill=c%2B%2B

## Фильтрация проектов по навыкам

Страница проектов:

/projects/list/

Пример фильтрации:

/projects/list/?skill=Python

Проекты фильтруются по необходимым навыкам проекта.

## Избранное

Авторизованный пользователь может добавлять проекты в избранное и удалять их из избранного.

Страница избранных проектов:

/projects/favorites/

## Участие в проектах

Авторизованный пользователь может присоединиться к чужому проекту или отказаться от участия.

Владелец проекта и администратор могут редактировать проект и завершать его.

## Кнопки «Поделиться»

В проекте реализованы:

* «Поделиться профилем»;
* «Поделиться ссылкой» на проект.

В браузерах на основе Chromium ссылка копируется в буфер обмена. Если браузер не разрешает копирование, ссылка выводится на экран.

## Частые проблемы

### Ошибка с psycopg или psycopg2

В проекте используется современный драйвер:

psycopg[binary]

Он должен быть указан в `requirements.txt`.

Переустановить зависимости можно командой:

pip install -r requirements.txt
